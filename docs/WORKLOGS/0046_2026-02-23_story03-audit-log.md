## Key Decisions

- Nil-guard `if r.Log != nil {` wraps every new log call, consistent with existing pattern
- For the cancel path (F), the audit log fires in the `else` branch of the delete error check —
  this correctly fires on both a clean delete (no error) and on an IsNotFound (already gone),
  matching the intent of "we successfully cancelled this job"
- For job succeeded/failed audit log, `rjob.Status.PRRef` is read after the status patch succeeds,
  so the logged value reflects whatever the agent wrote before the reconcile ran (may be empty
  if the agent hasn't patched it yet, which is expected and acceptable)
- No logic was changed; all existing tests pass unchanged
