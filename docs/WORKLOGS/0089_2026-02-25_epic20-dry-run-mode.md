## Key Decisions

- **Dry-run enforcement is deterministic** — wrappers (not prompt only) block writes. This is consistent with the threat model (AR-06 / AV-13). The prompt HARD RULE 11 is informational, not the enforcement mechanism.
- **`exec` removed from dry-run path in entrypoints** — shell must continue after opencode exits to call `emit_dry_run_report`. Normal path retains `exec` for correct exit code forwarding and no shell overhead.
- **git wrapper does not redact stdout** — the story explicitly preserved this distinction from epic12/epic25; only write subcommands are gated, not output.
- **`fetchDryRunReport` does not error-return** — if log fetch fails, a descriptive string is stored in `status.message` and the reconciler does not retry (the Job succeeded; the report is best-effort).
- **Truncation test uses preamble** — after code review identified that the naive `15_000 x` construction did not truly exercise LimitReader cutting post-sentinel content, the test was corrected to use `"preamble\n" + sentinel + "\n" + strings.Repeat("x", 11_000)`.
