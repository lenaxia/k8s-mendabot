## Key Decisions

### TC-02 and TC-03 sequential suppression limitation
The `pendingPeers()` function excludes jobs with `Phase != "" && Phase != Pending`.
This means once a primary dispatches (Phase=Dispatched), it's invisible to the secondary's
peer list. A single test sequence cannot assert BOTH "primary dispatched with group label"
AND "secondary suppressed". TC-02 and TC-03 test only the primary path, with a comment
explaining the design constraint.

### `dynamicFakeJobBuilder`
Added to `fakes_test.go` to generate unique Job names per RemediationJob UID,
preventing AlreadyExists errors when two jobs in the same namespace both dispatch.
