### 2. GAP DC-5 (Major) — `AllFindings` included unmatched peers from other groups

`CorrelationGroup.AllFindings` was assembled from `candidate + ALL peers` regardless of
which peers the rule actually matched. In a mixed-node scenario (`MultiPodSameNodeRule`
with pods on multiple nodes), findings from pods on unrelated nodes were injected into
`FINDING_CORRELATED_FINDINGS`.

**Fix:** Added `MatchedUIDs []types.UID` to `domain.CorrelationResult`. Each rule now
populates `MatchedUIDs` with exactly the jobs in its match. The correlator builds
`AllFindings` from only those UIDs. Fallback to all-peers when `MatchedUIDs` is empty
(preserves behavior for stub rules in tests).

Files: `internal/domain/correlation.go`, `internal/correlator/rules.go`,
`internal/correlator/correlator.go`, plus new tests in `correlator_test.go` and
`rules_test.go`.

### 3. GAP XC-2 (Major) — testdata CRD missing `enum` and `x-kubernetes-preserve-unknown-fields`

`testdata/crds/remediationjob_crd.yaml` was missing:
1. `enum` constraint on `status.phase` — envtest would accept invalid phase strings
2. `x-kubernetes-preserve-unknown-fields: true` on `conditions.items` — envtest was
   silently stripping `reason`, `message`, `lastTransitionTime` from condition objects
   on every status patch

**Fix:** Both fields added to match the production CRD exactly.

### 4. GAP OC-5 (Minor) — `Status.CorrelationGroupID` not set on primary jobs

Primary jobs had the `correlation-group-id` label set but `Status.CorrelationGroupID`
left empty — an asymmetry with secondary jobs. `kubectl get rjob` would show a blank
`CORRELATION-GROUP-ID` column for primaries.

**Fix:** Added a status patch in the primary dispatch path setting `CorrelationGroupID`.

### 5. GAP TC-1b, TC-2a, TC-3a, TC-6 (Minor) — Missing test assertions

- TC-1b: New test `TestMultiPodSameNodeRule_MultipleNodes_OnlyThresholdNodeMatches` —
  3 pods on node-abc (fires), 2 on node-def (below threshold), threshold=3
- TC-2a: `TestCorrelationWindow_SecondaryIsSuppressed` now asserts the `CorrelationGroupIDLabel` label is present
- TC-3a: `TestCorrelationWindow_PrimaryIsDispatched` now asserts the primary's own finding is in `CorrelatedFindings`
- TC-6: `TestCorrelationIntegration_TC02b_SecondaryIsSuppressed` now asserts the label on the suppressed job

### 6. GAP IC-6 (Minor) — Worklog 0047 contained incorrect documentation

Worklog 0047 claimed `pendingPeers()` "intentionally includes blank-phase jobs."
The code has always been `Phase == PhasePending` only (strict equality). The code is
correct; the description was wrong. Per append-only worklog rules, 0047 is not modified.
The correction is recorded here.

## Key Decisions

`MatchedUIDs` fallback to all-peers when empty: this preserves existing test behavior
for any rule that doesn't populate the field (e.g. simple unit-test stubs). Rules in
production code always populate it.

Status and labels are separate API subresources and require two separate patches. The
primary path now correctly issues a status patch before the label patch.
