## Pass 1 — Scripts and Workflows

### Finding 2026-02-27-003: `git` wrapper — extend dry-run blocklist

**File:** `docker/scripts/redact-wrappers/git`

Added to the `case` block inside the `_dry_run=true` guard:

- `reset|rm|clean|rebase` — blocked unconditionally (destructive working-tree ops)
- `config` — scanned for `--global`/`--system` flags; local config reads pass through
- `remote set-url` — blocked (attacker could redirect future pushes)

The `push`, `commit`, and annotated-`tag` blocks were retained unchanged.

### Finding 2026-02-27-004: `gh` wrapper — allowlist rewrite

**File:** `docker/scripts/redact-wrappers/gh`

Replaced the blocklist approach (which missed `gh api` entirely) with a fail-closed
allowlist. In dry-run mode, only the following are permitted:

| Command | Allowed sub-operations |
|---------|----------------------|
| `gh api` | No `--method`/`-X` flag, or explicitly `-X GET`/`--method GET` |
| `gh auth` | `status`, `token` |
| `gh repo` | `view`, `list`, `clone` |
| `gh pr` | `view`, `list`, `checks`, `diff`, `review` |
| `gh issue` | `view`, `list` |
| `gh run` | `view`, `list`, `watch` |
| `gh release` | `view`, `list`, `download` |
| `gh workflow` | `view`, `list` |

Everything else — including `gh api -X POST/PUT/PATCH/DELETE`, `gh secret set`,
`gh variable set`, `gh repo create/delete/edit` — is blocked with `exit 0`.

The three-layer dry-run detection (`/mendabot-cfg/dry-run` sentinel, `/proc/1/environ`,
`$DRY_RUN` env var) and the `redact` binary guard were preserved unchanged.

### Finding 2026-02-27-009: `emit_dry_run_report` — pipe through redact

**File:** `docker/scripts/entrypoint-common.sh`

In `emit_dry_run_report()`:

- `REPORT` now reads via `redact < /workspace/investigation-report.txt` instead of `cat`
- `PATCH` pipes the full `git diff HEAD` + untracked diff output through `redact` before
  assignment
- Added pre-flight: if `redact` is not in PATH, the function aborts with `exit 1` rather
  than silently falling back to unredacted output

### Finding 2026-02-27-013: `ai-comment.yml` — author_association gate

**File:** `.github/workflows/ai-comment.yml`

Added `github.event.comment.author_association` check to the `if:` condition of the
`respond` job. Only `OWNER`, `MEMBER`, and `COLLABORATOR` associations can trigger.
Unauthorized users are silently skipped (exit 0 / green workflow run, no noise).

Before:
```yaml
if: |
  startsWith(github.event.comment.body, '/ai') ||
  contains(github.event.comment.body, ' /ai')
```

After:
```yaml
if: |
  (startsWith(github.event.comment.body, '/ai') ||
   contains(github.event.comment.body, ' /ai')) &&
  (github.event.comment.author_association == 'OWNER' ||
   github.event.comment.author_association == 'MEMBER' ||
   github.event.comment.author_association == 'COLLABORATOR')
```

## Pass 2 — Go Types, CRD, Config, JobBuilder, Helm

### Findings 2026-02-27-005 + -006: `agentImage` and `agentSA` immutability

**File:** `api/v1alpha1/remediationjob_types.go`

Added two CEL XValidation rules to `RemediationJobSpec` (alongside existing immutability
rules for `fingerprint`, `sourceType`, `sinkType`):

```go
// +kubebuilder:validation:XValidation:rule="!has(oldSelf.agentImage) || self.agentImage == oldSelf.agentImage",message="spec.agentImage is immutable"
// +kubebuilder:validation:XValidation:rule="!has(oldSelf.agentSA) || self.agentSA == oldSelf.agentSA",message="spec.agentSA is immutable"
```

Note: value-allowlist validation (restricting to specific registries/SA names) was
deferred — it requires a Helm-configurable CEL expression to support custom registries
and SA names. The immutability rule closes the post-creation mutation attack vector
without locking out legitimate operator configurations.

## Pass 3 — Private Key Volume Mount

### Finding 2026-02-27-002: GitHub App private key in environment

**File:** `charts/mendabot/templates/deployment-watcher.yaml`

Within the `{{- if .Values.watcher.prAutoClose }}` block:

- Removed `GITHUB_APP_PRIVATE_KEY` env var (secretKeyRef)
- Added `volumeMounts` section to the watcher container:
  ```yaml
  volumeMounts:
  - name: github-app-private-key
    mountPath: /var/run/secrets/mendabot/github-app-private-key
    readOnly: true
  ```
- Added `volumes` section to the pod spec:
  ```yaml
  volumes:
  - name: github-app-private-key
    secret:
      secretName: github-app
      items:
      - key: private-key
        path: private-key
  ```

With `prAutoClose=false`, no volume or mount is generated (verified via helm template).

`GITHUB_APP_ID` and `GITHUB_APP_INSTALLATION_ID` remain as env vars — these are
numeric IDs with no standalone exploit value (cannot mint tokens without the private key).

**File:** `cmd/watcher/main.go`

Replaced:
```go
privKey, err := jwt.ParseRSAPrivateKeyFromPEM([]byte(os.Getenv("GITHUB_APP_PRIVATE_KEY")))
```

With:
```go
privKeyPEM, err := os.ReadFile("/var/run/secrets/mendabot/github-app-private-key/private-key")
if err != nil {
    logger.Fatal("failed to read GitHub App private key from volume — ...")
}
privKey, err := jwt.ParseRSAPrivateKeyFromPEM(privKeyPEM)
```

The key is no longer present in `/proc/1/environ` or in `kubectl get pod -o yaml` env dumps.
